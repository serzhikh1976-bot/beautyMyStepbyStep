# 📁 Структура проекта beautyMyStepbyStep

> Telegram-маркетплейс красоты: клиенты ищут мастеров, мастера регистрируют профили,
> между ними работает анонимный чат. Поддерживает несколько городов (несколько ботов).

---

## Корень проекта

| Файл | Назначение |
|------|-----------|
| `.env` | Секретные переменные окружения (токены, URL Supabase, ID суперадмина). **Никогда не коммитить в git.** |
| `.env.example` | Шаблон `.env` — показывает какие переменные нужны, но без реальных значений |
| `.gitignore` | Что не попадает в git (node_modules, .env и т.д.) |
| `package.json` | Зависимости и npm-скрипты (`dev`, `build`, `register-webhooks`) |
| `tsconfig.json` | Настройки TypeScript компилятора |

## `src/` — исходный код

### 🚀 Точка входа и сервер

#### `src/index.ts`
**Запускает всё приложение.**
- Загружает `.env`
- Создаёт Fastify-сервер через `buildServer()`
- Запускает его на нужном порту
- Обрабатывает сигналы `SIGINT`/`SIGTERM` для graceful shutdown

#### `src/server.ts`
**Строит HTTP-сервер (Fastify).**
- Регистрирует admin web-панель
- Обрабатывает `GET /` — health check (Railway/VPS проверяют что сервер жив)
- Обрабатывает `POST /webhook/:uuid` — **единственная точка входа для всех Telegram-обновлений**. UUID в URL определяет какой бот получил сообщение

#### `src/db.ts`
**Подключение к базе данных Supabase.**
- Создаёт и экспортирует клиент `db` для всех запросов к БД
- Экспортирует интерфейс `BotRecord` — структура записи из таблицы `bots`

#### `src/register-webhooks.ts`
**Одноразовый скрипт (запускается вручную через `npm run register-webhooks`).**
- Читает все активные боты из БД
- Регистрирует вебхук у Telegram для каждого бота
- Регистрирует вебхук для админ-бота отдельно

### 🤖 `src/bot/` — логика City-ботов

Это боты для конкретных городов (Одесса, Киев и т.д.). Каждый живёт в одном процессе, но изолирован через кэш инстансов.

#### `src/bot/cache.ts`
**Кэш инстансов ботов в памяти.**
- `getBot(uuid)` — возвращает готовый инстанс бота по UUID; если нет — создаёт его, загружая данные из БД и кэшируя
- `invalidateBot(uuid)` — удаляет бот из кэша (при выключении города через админку)
- `cacheSize()` — сколько ботов сейчас активно

#### `src/bot/factory.ts`
**Фабрика: создаёт инстанс бота для конкретного города.**
- Подключает middleware: сессии, сцены (WizardScene)
- Регистрирует все хэндлеры: start, search, chat, profile
- Устанавливает глобальный `bot.catch()` — перехватывает необработанные ошибки

#### `src/bot/helpers.ts`
**Общие утилиты для ботов.**
- `clearButtons(bot, ctx)` — убирает inline-кнопки с уже нажатого сообщения (защита от повторного нажатия)
- `sendTracked(bot, chatId, side, recipientId, send)` — отправляет сообщение и запоминает его ID; при следующей отправке автоматически убирает кнопки с предыдущего сообщения (чтобы нельзя было случайно нажать старую кнопку «Забанить» из истории)

#### `src/bot/keyboards.ts`
**Определения клавиатур.**
- `masterKeyboard` — Reply-клавиатура мастера (кнопки «Мой профиль» и т.д.)
- `endChatKeyboard(chatId)` — кнопка завершения чата для клиента
- `masterActionsKeyboard(chatId)` — кнопки мастера в чате (Завершить + Забанить)

#### `src/bot/roles.ts`
**Сохранение роли пользователя.**
- `saveRole(botId, telegramId, role)` — upsert в таблицу `users`: записывает или обновляет роль (`client` | `master`) для данного пользователя в данном боте

### 📋 `src/bot/handlers/` — обработчики команд и событий

#### `src/bot/handlers/start.ts`
**Обработка старта бота и выбора роли.**
- `/start` — проверяет есть ли уже роль у пользователя; если да — показывает нужную клавиатуру; если нет — предлагает выбрать «Клиент» или «Мастер»
- `role:client` — сохраняет роль клиента, показывает кнопку поиска
- `role:master` — сохраняет роль мастера, запускает сцену регистрации

#### `src/bot/handlers/profile.ts`
**Управление профилем мастера.**
- `showMasterProfile()` — формирует и отправляет карточку профиля (фото + данные + кнопки управления)
- Кнопка «Мой профиль» — показывает профиль
- `edit:photos` / `edit:services` / `edit:district` / `edit:price` — запускают соответствующие сцены редактирования
- `toggle:active` — переключает статус «Активен / На паузе» (когда мастер на паузе — его не видят клиенты)

#### `src/bot/handlers/search.ts`
**Поиск мастеров клиентом.**
- Кнопка «Найти мастера» / команда `/search` — проверяет нет ли уже активного чата, потом запускает визард выбора услуги → района → мастера
- `master_card:` — показывает полную карточку мастера с фото и кнопкой «Написать»

#### `src/bot/handlers/chat.ts`
**Весь анонимный чат между клиентом и мастером.**
- `chat:` — открывает новый чат (проверяет: нет ли уже чата, активен ли мастер, не забанен ли клиент)
- `end_chat:` → `end_chat_confirm:` — завершение чата с подтверждением
- `end_chat_cancel:` — отмена завершения
- `ban_client:` → `ban_client_confirm:` — бан клиента мастером с подтверждением
- `bot.on('text')` — маршрутизирует текстовые сообщения между участниками чата
- `bot.on('photo')` — маршрутизирует фото между участниками чата
- `routeChatMessage()` — внутренняя функция: определяет кто пишет (клиент или мастер) и пересылает сообщение нужному получателю
- `noActiveChatMessage()` — формирует правильный ответ если у пользователя нет активного чата

### 🎬 `src/scenes/` — мультишаговые сцены (WizardScene)

Сцены — это пошаговые диалоги, где бот «помнит» на каком шаге находится пользователь.

#### `src/scenes/master-registration.ts`
**Регистрация нового мастера (5 шагов).**
1. Имя
2. Выбор услуг (мультиселект с чекбоксами)
3. Выбор района (+ подрайон если есть)
4. Минимальная цена
5. Фото портфолио (до 5 штук, `/done` для завершения)

В конце: сохраняет профиль в `masters_profiles`, услуги в `master_services`, активирует 30-дневный триал.

#### `src/scenes/client-search.ts`
**Поиск мастера клиентом (2 шага).**
1. Выбор района (показывает только где есть мастера с нужной услугой)
2. Выбор подрайона (если есть) → список мастеров

Экспортирует `getServicesWithMasters()` — используется в `handlers/search.ts` для отображения только тех услуг, где есть хотя бы один активный мастер.

#### `src/scenes/edit-photos.ts`
**Редактирование фото портфолио мастером.**
Принимает новые фото (до 5), `/done` — сохраняет, `/skip` — удаляет все фото.

#### `src/scenes/edit-price.ts`
**Редактирование минимальной цены мастером.**
Один шаг: вводит число → сохраняет в профиле.

#### `src/scenes/edit-district.ts`
**Редактирование района мастером.**
Выбор района → выбор подрайона (если есть) → сохранение.

#### `src/scenes/edit-services.ts`
**Редактирование списка услуг мастером.**
Мультиселект (чекбоксы) → «Готово» → сохраняет.

### 🔧 `src/shared/` — общие утилиты

#### `src/shared/telegram-html.ts`
**Экранирование HTML для Telegram.**
- `escapeHtml(str)` — заменяет `<`, `>`, `&` на HTML-сущности. Используется везде где в сообщение подставляются пользовательские данные (имена, тексты) — защита от инъекций в HTML-разметку Telegram.

#### `src/shared/districts.ts`
**Общие функции работы с районами.**
- `getDistricts(botId)` — загружает районы из БД для конкретного бота
- `buildListKeyboard(items, prefix)` — строит InlineKeyboard из списка районов/подрайонов

### 👑 `src/admin/` — Веб-панель администратора

Отдельный бот + веб-интерфейс для суперадмина. Доступен по `/admin/...`.

#### `src/admin/bot.ts`
**Telegram-бот суперадмина.**
- Singleton: один инстанс на весь процесс, конфиг из `.env`
- Пускает только суперадмина (проверяет `SUPER_ADMIN_ID`)
- `/start` → дашборд со статистикой по всем городам
- Кнопки: детали по городу, список активных чатов

#### `src/admin/web.ts`
**Регистрация всех HTTP-маршрутов веб-панели.**
- Подключает маршруты из `chats.ts` и `settings.ts`
- Настраивает Basic Auth (логин/пароль из `.env`)
- Добавляет парсинг форм (`application/x-www-form-urlencoded`) для POST-запросов

#### `src/admin/shared.ts`
**Общие утилиты веб-панели.**
- `layout(title, body, opts)` — HTML-шаблон страницы (шапка, навигация, стили)
- `requireAuth(request, reply)` — проверка Basic Auth
- `escapeHtml()` — экранирование для HTML-страниц

#### `src/admin/chats.ts`
**Страницы чатов, клиентов и банов в веб-панели.**

| Маршрут | Что делает |
|---------|-----------|
| `GET /admin/chats` | Список всех чатов с фильтрами (город, статус, ID клиента/мастера) |
| `GET /admin/chats/:id` | Полная переписка одного чата с фото |
| `GET /admin/photo/:botId/:fileId` | Прокси фото из Telegram (не светит токен бота на клиенте) |
| `GET /admin/clients` | Список уникальных клиентов |
| `GET /admin/bans` | Список банов |
| `POST /admin/bans/:id/unban` | Разблокировать клиента |

#### `src/admin/settings.ts`
**Страницы управления ботами, услугами и районами в веб-панели.**

| Маршрут | Что делает |
|---------|-----------|
| `GET /admin/bots` | Список всех городов/ботов |
| `POST /admin/bots` | Создать новый бот (город) + авторегистрация вебхука |
| `POST /admin/bots/:id/toggle` | Включить / выключить бот |
| `GET /admin/services` | Услуги (с фильтром по городу) |
| `POST /admin/services` | Добавить услугу |
| `POST /admin/services/toggle` | Включить/выключить услугу для города |
| `GET /admin/districts` | Список районов города |
| `POST /admin/districts` | Добавить район |
| `GET /admin/districts/:id` | Подрайоны конкретного района |
| `POST /admin/districts/:id/delete` | Удалить район |
| `POST /admin/districts/:id/sub-districts` | Добавить подрайон |
| `POST /admin/sub-districts/:id/delete` | Удалить подрайон |

## Схема взаимодействия

```
Telegram → POST /webhook/:uuid
               │
               ├─ uuid == ADMIN_BOT_UUID  →  admin/bot.ts
               │
               └─ uuid == city bot       →  bot/cache.ts → bot/factory.ts
                                                              │
                                              ┌──────────────┼──────────────┐
                                         start.ts      search.ts       chat.ts
                                                                        │
                                                               scenes/*.ts (WizardScene)
```
